## Story ID
   * ID Format: US_006

## Story Title
   * Inactive Employee Auto-Disable from Workday

## Description
  * As a system administrator, I want terminated or inactive employees automatically disabled within 4 hours of their Workday employment status change, with all active sessions revoked and historical data preserved for audit compliance, so that platform access is revoked promptly and securely for departed employees.

## Acceptance Criteria
  1. **Given** the Workday sync detects an employee with EmploymentStatus changed to 'Inactive' or 'Terminated', **When** the sync processes this record, **Then** the system sets the user's account status to 'Inactive' with the DeactivatedDate timestamp and logs the deactivation in the AuditLog.

  2. **Given** an employee is deactivated, **When** the deactivation is processed, **Then** all active sessions for that user are immediately invalidated (Session table updated, JWT blacklisted) so the user cannot access any platform features.

  3. **Given** a deactivated employee attempts to log in via SSO, **When** the authentication callback is received, **Then** the system checks the user's active status, blocks access with the message "Your account has been deactivated. Contact your administrator.", and logs the blocked attempt.

  4. **Given** an employee is deactivated, **When** the deactivation completes, **Then** all historical data (credit transactions, course enrollments, badges, audit logs) is preserved with soft-delete pattern; no data is hard-deleted.

  5. **Given** an employee is reactivated in Workday (e.g., return from leave), **When** the next sync detects the status change back to 'Active', **Then** the system reactivates the user account with all historical data intact and sends a "Welcome back" notification.

## Edge Cases
   * What happens when a deactivated user has pending certification applications? (Applications cancelled with reason "Employee deactivated"; manager notified; no credits allocated)
   * How does the system handle deactivation during an active course session? (Session immediately terminated; progress preserved at last auto-save; enrollment status set to 'Dropped')
   * What happens if Workday sync fails and deactivation is delayed beyond 4 hours? (Sync retry mechanism triggers within 1 hour; admin alerted; manual deactivation available via admin console as fallback)
