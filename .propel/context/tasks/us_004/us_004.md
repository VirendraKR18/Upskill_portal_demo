## Story ID
   * ID Format: US_004

## Story Title
   * User Profile View and Update

## Description
  * As a learner, I want to view and update my profile information including bio, skills, interests, and notification preferences, so that my profile accurately reflects my current capabilities, goals, and communication preferences.

## Acceptance Criteria
  1. **Given** I am authenticated, **When** I navigate to my profile page, **Then** the system displays my current profile data: name, email, department, manager (read-only from Workday), and editable fields (bio, skills, interests, notification preferences).

  2. **Given** I am on my profile page, **When** I update my bio, skills, or interests and click "Save", **Then** the changes are persisted immediately, the UI confirms success via a toast notification, and the previous values are recorded in the change history.

  3. **Given** I update my notification preferences, **When** I toggle email notifications or in-app notifications, **Then** the preference is saved and takes effect for all future notifications without requiring logout.

  4. **Given** mandatory profile fields are defined, **When** I attempt to save with missing required fields, **Then** the form displays inline validation errors with specific, actionable messages (e.g., "Bio is required and must be between 10-500 characters").

  5. **Given** profile data is stored, **When** it is persisted to the database, **Then** sensitive fields are encrypted at rest using AES-256 per NFR-005.

  6. **Given** I view my profile, **When** I check the change history section, **Then** I see a timestamped log of all previous profile updates (field changed, old value, new value, date).

## Edge Cases
   * What happens when two sessions update the same profile simultaneously? (Optimistic concurrency control via row versioning; last write wins with conflict notification)
   * How does the system handle extremely long text in bio field? (Server-side validation enforces 500 character limit; API returns 422 with specific error)
   * What happens when a Workday sync updates read-only fields while user is editing? (Editable fields preserved; read-only fields updated silently; user notified if page stale)
